Load Balancer + Auto Scaling: la combinación ganadora

¿Qué hace cada uno?

  • Load Balancer (ALB/NLB) — distribuye el tráfico entrante entre instancias del target group. Hace health checks y deja de enviar tráfico a instancias unhealthy. Pero eso es todo: no crea ni destruye nada.
  • Auto Scaling Group (ASG) — mantiene un número deseado de instancias EC2. Reacciona a métricas (CPU, requests) para hacer scale out o scale in. Pero solo no sabe cómo distribuir el tráfico.

Juntos forman la dupla estándar de resiliencia y elasticidad.

Cómo se integran (el flujo)

  1. El ALB hace un health check a cada instancia (HTTP GET a un endpoint, configurable).
  2. Si una instancia falla → el ALB la saca del target group → deja de recibir tráfico.
  3. El ASG recibe esa señal de salud (usa los health checks del ALB si está configurado así) → termina la instancia unhealthy → lanza una nueva.
  4. La nueva instancia pasa el health check → el ALB la agrega al target group → empieza a recibir tráfico.
  5. Si el CPU promedio del ASG sube al 70% → el scaling policy dispara un scale out → el ALB automáticamente empieza a enviarle tráfico a las nuevas instancias.

Regla clave del examen: el ASG necesita saber que use los health checks del ELB, no solo los de EC2. Si no, puede no reemplazar una instancia que falla a nivel aplicación pero sigue "viva" a nivel EC2.

Comparación para el examen

Solo Load Balancer Load Balancer + ASG
Distribución de tráfico ✅ Sí ✅ Sí
Excluye instancias unhealthy ✅ Sí ✅ Sí
Reemplaza instancias unhealthy ❌ No ✅ Automático
Escala ante picos de tráfico ❌ No ✅ Automático
Maneja caída de una AZ ⚠️ Parcial ✅ Completo
Operación necesaria en fallos Manual Ninguna

Trampas de examen

Trampa 1 "El Load Balancer protege contra fallos" → parcialmente verdad. Redirige el tráfico, pero sin ASG la instancia muerta queda muerta y la capacidad se reduce.
Trampa 2 Un ASG con health check type = EC2 (default) NO reemplaza una instancia que falla a nivel aplicación (proceso caído, 500 errors). Necesitás health check type = ELB para eso.
Trampa 3 Si el enunciado dice "highly available" + "fault tolerant" → siempre es Multi-AZ + ASG + ALB. Un solo AZ con ASG no es suficiente.

Mnemónico

"El ALB decide; el ASG actúa."

El ALB sabe quién está sano; el ASG repara y escala.
